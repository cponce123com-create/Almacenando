MEJORA 4 — Preferencias funerarias que persisten correctamente
Archivo: artifacts/legado/src/pages/funeral/index.tsx
El problema es que el formulario se inicializa con defaultValues vacíos y el useEffect que hace form.reset(prefs) corre después, pero si el usuario navega a la página y los datos ya están en caché de React Query, el reset no se dispara de nuevo.
Fix: Cambiar el useEffect para que use form.reset() con los valores de prefs usando esta lógica:
typescriptconst { data: prefs, isLoading } = useGetFuneralPreferences();

const form = useForm({
  defaultValues: {
    burialType: prefs?.burialType || "",
    ceremonyType: prefs?.ceremonyType || "",
    musicNotes: prefs?.musicNotes || "",
    dressCode: prefs?.dressCode || "",
    guestNotes: prefs?.guestNotes || "",
    locationNotes: prefs?.locationNotes || "",
    additionalNotes: prefs?.additionalNotes || "",
  }
});

useEffect(() => {
  if (prefs) {
    form.reset({
      burialType: prefs.burialType || "",
      ceremonyType: prefs.ceremonyType || "",
      musicNotes: prefs.musicNotes || "",
      dressCode: prefs.dressCode || "",
      guestNotes: prefs.guestNotes || "",
      locationNotes: prefs.locationNotes || "",
      additionalNotes: prefs.additionalNotes || "",
    });
  }
}, [prefs?.burialType, prefs?.ceremonyType, prefs?.musicNotes, prefs?.dressCode, prefs?.guestNotes, prefs?.locationNotes, prefs?.additionalNotes]);
La dependencia del useEffect debe ser cada campo individual, no el objeto prefs completo, para que React detecte el cambio correctamente.
