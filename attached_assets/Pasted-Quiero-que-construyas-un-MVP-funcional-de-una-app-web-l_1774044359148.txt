Quiero que construyas un MVP funcional de una app web llamada temporalmente “Legado”, enfocada en legado digital póstumo.

La app debe permitir que una persona deje contenido para sus familiares o destinatarios, y que ese contenido solo se entregue cuando se confirme su fallecimiento mediante un flujo de verificación.

Quiero una versión moderna, limpia, emocional pero profesional. No debe sentirse lúgubre ni fúnebre; debe sentirse cálida, segura, elegante y confiable.

========================
1. OBJETIVO DEL MVP
========================

El MVP debe permitir:

1. Registro e inicio de sesión de usuarios.
2. Perfil básico del usuario.
3. Crear y gestionar contenido de legado:
   - video
   - carta
   - audio
   - foto
   - documento
   - nota/instrucciones de sepelio
4. Crear y gestionar destinatarios.
5. Crear y gestionar contactos de confianza.
6. Configurar reglas básicas de activación.
7. Flujo de reporte y confirmación de fallecimiento.
8. Panel admin mínimo para aprobar liberación.
9. Portal privado para que los destinatarios vean solo el contenido asignado.
10. Subida de archivos con Cloudinary.
11. Base de datos en Neon.
12. Despliegue pensado para Render.

========================
2. STACK TÉCNICO
========================

Quiero que uses este stack:

- Next.js 14+ con App Router
- TypeScript
- Tailwind CSS
- Prisma ORM
- PostgreSQL compatible con Neon
- Cloudinary para archivos
- Auth.js o autenticación propia simple con email/password y sesiones seguras
- Zod para validaciones
- React Hook Form donde ayude
- Render como target de deploy

Si necesitas elegir entre simplicidad y complejidad, prioriza simplicidad, claridad y mantenibilidad.

========================
7. REGLAS DE NEGOCIO
========================

Implementa estas reglas:

1. Un item de legado no puede liberarse si no tiene al menos un destinatario asignado.
2. El usuario no puede dejar su legado como “activo” si no tiene:
   - al menos 1 destinatario
   - al menos 2 contactos de confianza
3. Un death report necesita mínimo 2 confirmaciones para pasar a revisión.
4. En el MVP, toda liberación final requiere aprobación admin.
5. Cada destinatario solo ve los items asignados a él.
6. Los tokens de acceso deben expirar.
7. Toda acción importante debe dejar audit log.
8. El usuario debe poder guardar items en borrador.
9. El dashboard debe mostrar progreso/completitud del legado.
